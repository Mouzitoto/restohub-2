## Фаза 2: Аутентификация и авторизация

## Фаза 3: Admin API - Базовый функционал для ресторанов
### [Шаг 3.1: Рестораны](steps/phase-3/3.1-restaurant-management.md)
CRUD операции для ресторанов, настройка персональной страницы и контактной информации.

### [Шаг 3.2: Категории меню](steps/phase-3/3.2-menu-categories.md)
CRUD для категорий блюд, управление порядком отображения.

### [Шаг 3.3: Меню (блюда)](steps/phase-3/3.3-menu-items.md)
CRUD для блюд, управление ценами, описаниями и фотографиями.

### [Шаг 3.4: Этажи](steps/phase-3/3.4-floors.md)
CRUD для этажей ресторана.

### [Шаг 3.5: Залы (rooms)](steps/phase-3/3.5-rooms.md)
CRUD для помещений на этажах, загрузка планов помещений.

### [Шаг 3.6: Столы](steps/phase-3/3.6-tables.md)
CRUD для столов, настройка характеристик столов и загрузка фотографий.

### [Шаг 3.7: Бронирования и предзаказы](steps/phase-3/3.7-bookings-preorders.md)
API для просмотра и управления бронированиями и предзаказами, подтверждение через WhatsApp/Telegram.

### [Шаг 3.8: Подписки](steps/phase-3/3.8-subscriptions.md)
Реализация системы подписок: хранение информации о подписке и статус активности.

### [Шаг 3.9: Акции](steps/phase-3/3.9-promotions.md)
CRUD для промо-событий ресторанов (акции, тематические вечера, новинки).

### [Шаг 3.10: Аналитика](steps/phase-3/3.10-analytics.md)
API для получения статистики: бронирования, предзаказы, клиенты, экспорт данных.

### [Шаг 3.11: Клиенты](steps/phase-3/3.11-clients.md)
API для просмотра информации о клиентах, истории бронирований и предзаказов.

### [Шаг 3.12: Статусы бронирований](steps/phase-3/3.12-booking-statuses.md)
CRUD для управления статусами бронирований и предзаказов. Доступ только для ADMIN.

### [Шаг 3.13: Управление пользователями](steps/phase-3/3.13-user-management.md)
CRUD операции для управления пользователями системы (ADMIN и MANAGER). Доступ только для ADMIN.
