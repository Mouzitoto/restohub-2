# Шаг 4.9: Подписки

## Описание
Интерфейс для просмотра информации о подписке ресторана.

## Задачи

### 1. Страница информации о подписке

#### 1.1. Компонент SubscriptionPage
**Маршрут:** `/subscription` (защищенный)

**Структура:**
- Заголовок: "Подписка"
- Карточка с информацией о подписке
- Предупреждения (если есть)
- Кнопка "Продлить подписку" (только для ADMIN)

#### 1.2. Отображение информации о подписке
**Компонент:** SubscriptionCard

**Информация:**
- Тарифный план (название, описание, цена)
- Дата начала: `startDate`
- Дата окончания: `endDate`
- Дней осталось: `daysRemaining` (вычисляется)
- Статус: `isActive` (badge с цветом)
- Предупреждение: `isExpiringSoon` (если `daysRemaining <= 7`)

**Визуальная индикация:**
- Активна: зеленый badge
- Неактивна: серый badge
- Истекает скоро: оранжевый badge + предупреждение
- Просрочена: красный badge + предупреждение

**Логика загрузки:**
- GET запрос на `/admin-api/r/:id/subscription`
- При отсутствии подписки → показать сообщение "Подписка не активирована"

#### 1.3. Предупреждения
**Компонент:** SubscriptionAlerts

**Условия показа:**
1. Если `isExpiringSoon === true`:
   - Предупреждение: "Подписка истекает через {daysRemaining} дней"
   - Кнопка "Связаться для продления" (если MANAGER)
2. Если `isActive === false`:
   - Предупреждение: "Подписка неактивна"
   - Информация о причине (если есть `description`)
3. Если подписка просрочена:
   - Предупреждение: "Подписка просрочена"

#### 1.4. Продление подписки (только ADMIN)
**Компонент:** ExtendSubscriptionModal

**Форма:**
- Поле "Дата окончания" (date picker)
- Поле "Причина продления" (textarea, опционально)
- Кнопки: "Отмена" и "Продлить"

**Логика:**
- PUT запрос на `/admin-api/r/:id/subscription` с `endDate`
- При успехе → обновить информацию о подписке

### 2. Интеграция с API
- `GET /admin-api/r/:id/subscription` - получение информации
- `PUT /admin-api/r/:id/subscription` - обновление (только ADMIN)
