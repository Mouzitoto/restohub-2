# Шаг 6.2: Страница ресторана

## Описание
Реализация страницы ресторана с персонализированным дизайном, информацией о ресторане, меню и картой столов.

## Задачи

### 1. Информация о ресторане
- Отображение с применением персонализированного дизайна:
  - Фоновое изображение
  - Логотип
  - Цветовая тема ресторана
- Основная информация:
  - Название
  - Адрес
  - Телефон
  - Email
  - Описание
  - Кнопка на карту (Google Maps/Yandex Maps ссылка)

### 2. Меню с фото и описаниями
- Отображение категорий блюд
- Группировка блюд по категориям
- Для каждого блюда:
  - Фото (основное)
  - Название
  - Описание
  - Цена
  - Вес/объем (если есть)
  - Состав/ингредиенты (если есть)
- Фильтрация по категориям (tabs или menu)
- Скролл к категории при клике
- Возможность добавления блюда в предзаказ (если реализована форма предзаказа)

### 3. Кнопка бронирования
- Кнопка "Забронировать стол" (крупная, заметная)
- При клике → редирект на страницу выбора залов (`/r/:id/booking/rooms`)
- Расположение: в верхней части страницы или в sticky header

### 4. Акции и тематические вечера
**Логика загрузки:**
- GET запрос на `/client-api/r/:id/promotion?isCurrent=true` для получения текущих активных промо-событий
- Фильтрация по типу промо-события (опционально)

**Отображение промо-событий:**
- Для каждого промо-события:
  - Название (title)
  - Изображение (через компонент `ImagePreview` с `imageId`)
  - Описание (description, если есть)
  - Тип промо-события (badge: "Акция", "Тематический вечер", "Новинка" и т.д.)
  - Период действия:
    - Для обычных событий: "с DD.MM.YYYY по DD.MM.YYYY"
    - Для повторяющихся событий без endDate: "с DD.MM.YYYY (бесконечно)"
    - Для повторяющихся событий с endDate: "с DD.MM.YYYY по DD.MM.YYYY"
  - Информация о повторении (если `isRecurring=true`):
    - Если `recurrenceType='WEEKLY'` и `recurrenceDayOfWeek` указан: "Каждый [день недели]" (например, "Каждый четверг")
    - Если `recurrenceType='DAILY'`: "Ежедневно"
    - Если `recurrenceType='MONTHLY'`: "Ежемесячно"
  - Индикатор повторяющегося события (иконка/бейдж "Повторяется", если `isRecurring=true`)
- Группировка по типу (опционально): акции отдельно, тематические вечера отдельно
- Карточки или список промо-событий

### 5. Применение персонализированного дизайна
- Использование цветовой темы ресторана (CSS variables)
- Применение кастомных стилей (если указаны)
- Отображение фонового изображения
- Отображение логотипа

### 6. Navigation
- Якорные ссылки для быстрого перехода:
  - О ресторане
  - Меню
  - Столы
  - Акции
- Sticky header при скролле (опционально)

### 7. Responsive дизайн
- Адаптивность для мобильных устройств
- Оптимизация изображений для разных экранов
- Touch-friendly интерфейс

### 8. UX улучшения
- Loading states
- Error handling
- Smooth scrolling
- Lazy loading изображений
